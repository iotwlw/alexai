1. 图片下载按钮

原实现固定创建大图和缩略图两个按钮，并用“直接显示 / 不显示”控制。新实现把“何时显示”和“显示哪些清晰度”拆成两个独立设置。

默认策略

displayMode: hover、qualityMode: high。默认页面更安静，只在悬停图片时出现 HD；大幅视频入口保持可见。

性能逻辑

悬停使用根节点 class 与 CSS :hover/:focus-within，不为每张图片增加 mouseenter/mouseleave 监听。

节点控制

仅高清模式不会创建 SD 按钮；切换到“高清 + 标清”后才重建需要的控件。

旧设置迁移

历史 hidden 设置自动映射为 hover，避免升级后按钮永久不可见。

连续修改保护

页面入口先更新内存状态，再按顺序写入 Storage；清晰度或启用状态改变时立即清理旧控件并重扫，避免快速操作把 SD 状态覆盖或遗留在 DOM。

  • HD 使用现有高分辨率候选解析逻辑，选择实际可加载的较大图片。
  • SD 使用页面当前 currentSrc/src/data-src，不执行放大。
  • 图片按钮仍为绝对定位，不改变 Amazon 图片容器尺寸和页面布局。
  • Popup 与页面小图标写入同一个 imageDownloadSettings，不存在两套状态。

2. 大幅视频下载入口

小视频和缩略卡继续使用紧凑的 VID；大幅视频根据明确的 hero 宿主选择器，或宽度至少 360px 且高度至少 180px 的尺寸规则识别。

视频宿主匹配 -> 大幅视频？ -> 是：88 x 38，显示“下载视频” -> 否：保留紧凑 VID -> 下载中：统一显示进度或省略号 -> 完成：恢复对应宿主的原标签

这样提高主视频入口的发现率，同时避免每个缩略视频按钮都变大并遮挡内容。

3. Alexa / Rufus 抓取高级版

原实现把“链接巡查”错误理解成另一套 Amazon 商品巡检，并接入 /api/asin-inspection。现已删除该工作区、结果表和 API 集成；用户所说的链接巡查就是当前扩展已有的 Alexa / Rufus 批量抓取。

Popup 输入授权码 -> 请求授权服务的可选 host permission -> Background 生成/读取随机 deviceId -> POST /v1/licenses/activate -> 服务端返回 features: ["alexa_scraping"] -> Popup 开放“开始高级抓取” -> 每次开始/继续前 Background 再次在线验证
双层门槛

Popup 未授权时禁用按钮；即使绕过界面直接发消息，Background 仍会在创建 Amazon 标签页前验证授权。

网络边界

HTTP 只允许 localhost 和 127.0.0.1；远程服务必须使用 HTTPS。

权益边界

任意非空授权码不能解锁；响应必须有效、未过期、设备匹配，并明确包含 alexa_scraping。

隐私边界

授权请求只提交授权码、随机设备 ID 和扩展版本，不上传 Amazon Cookie、账号、URL、ASIN 或抓取结果。

能力边界 当前 MVP 在开始/继续时严格在线复验，尚未实现签名令牌、离线宽限和设备解绑。扩展本地代码可被查看或修改，授权不能形成不可绕过的 DRM；需要更强保护时，应把关键高级能力或持续更新放到服务端。

4. 交互与信息架构

  • Popup 顶部保留品牌、版本和当前免费版/专业版状态。
  • 只保留一个 Alexa / Rufus 抓取工作区，顶部直接说明“图片/视频免费，批量抓取需授权”。
  • 抓取设置作为可折叠次级区域，输入与开始按钮保持主路径。
  • 状态、统计和导出集中在右侧上下文区，减少视线往返。
  • 去除装饰性渐变和大量 Emoji，用单一蓝色主操作、绿色成功、琥珀色专业版状态建立一致层级。
  • 按钮、输入框和固定格式控件都有稳定尺寸，任务状态变化不会导致布局跳动。

5. 页面小图标设置

原来的 alexai Ready 状态块改为 42x42 的 A 品牌入口，搜索页和商品详情页均可使用。

样式隔离

面板使用 Shadow DOM，Amazon 页面 CSS 不会改变开关、选择器和按钮尺寸。

同步方式

所有设置写入 chrome.storage.local；页面入口会先更新当前页并串行保存，Popup 与 storage change 监听负责跨界面同步。

可访问性

支持键盘焦点、Esc 关闭、ARIA 标签和 :focus-visible。

页面干扰

面板默认关闭，只保留小图标；点击外部自动收起，不持续占用 Amazon 内容区域。
